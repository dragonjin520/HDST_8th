## 문제 정의
주어진 데이터는 택시 운전기사가 택시를 운전해서 생성해낸 데이터이다. 즉 사람이 생성한 데이터이고 사람마다, 시간마다 특성이 다를 것이다.


그런데 만약 이 데이터를 자율주행차가 만들었다면?

## Data Product 정의

Data Product는 단순히 데이터를 분석해 그래프나 통계를 만드는 것이 아니라, 수집·가공한 데이터를 이용해 특정 사용자가 반복적으로 의사결정을 내리거나 행동할 수 있도록 만든 제품 또는 기능이다.

```text
데이터 수집
→ 데이터 가공
→ 수요·비용·수익 지표 생성
→ 사용자에게 결과 제공
→ 실제 차량 배치 또는 업무 전환
```

외부 구매자에게 판매하는 경우도 있지만, 회사 내부 운영 효율화를 목적으로 사용하는 시스템 역시 Data Product에 해당한다.

이번 과제에서는 자율주행차 운영사가 차량 한 대의 가용시간당 평균 수익을 높일 수 있도록, 시간대별로 택시와 택배 업무를 전환하도록 지원하는 데이터 프로덕트를 구상했다.

## 일반 택시와 자율주행차의 차이

일반 택시와 자율주행차의 가장 큰 차이는 운전과 판단의 주체가 사람에서 시스템으로 바뀐다는 점이다.

| 구분 | 일반 택시 | 자율주행차 |
|---|---|---|
| 운전 주체 | 택시 운전사 | 자율주행 시스템 |
| 주변 환경 인식 | 운전자의 시각·청각·경험 | 카메라·라이다·레이더·GPS |
| 판단 방식 | 경험과 직관 | 알고리즘과 AI 모델 |
| 차량 배치 | 운전자가 직접 선택 | 중앙 시스템이 수요를 분석해 자동 배치 |
| 운행 가능 시간 | 운전자 근무시간에 영향 | 충전·정비 시간을 제외하면 지속 운행 가능 |
| 주요 데이터 | 승하차·거리·요금 등 운행 결과 | 운행 결과 + 센서·판단·제어·차량 상태 |

일반 택시는 주로 운행이 끝난 뒤 결과 데이터를 남긴다.

```text
언제 승객을 태웠는가
어디에서 타고 내렸는가
얼마나 이동했는가
얼마를 결제했는가
```

반면 자율주행차는 운행 결과뿐 아니라 다음과 같은 과정 데이터도 지속적으로 수집할 수 있다.

- 실시간 차량 위치
- 배터리 잔량과 예상 주행 가능 거리
- 센서 상태
- 객체 인식 결과
- 급제동·경로 변경 기록
- 자율주행 해제 및 원격 개입 기록
- 공차 이동 거리
- 현재 수행 중인 업무 유형

즉, 일반 택시 데이터가 운행 결과를 관리하기 위한 데이터라면 자율주행차 데이터는 차량의 판단과 운영을 직접 움직이는 데이터라고 볼 수 있다.

## 비즈니스 아이디어: 멀티서비스 자율주행차

자율주행 택시는 출퇴근 시간처럼 수요와 수익성이 높은 시점이 존재하지만, 그 외 시간에는 차량이 대기하면서 자원이 낭비될 수 있다.

따라서 택시 수요가 낮은 시간대를 도출한 뒤, 해당 시간 동안 택배나 배달 같은 다른 경제활동을 수행하도록 한다.

```text
07:00~11:00  택시
11:00~14:00  배달·택배
17:00~20:00  택시
02:00~05:00  허브 간 물류
```

고정된 시간표를 그대로 적용하기보다는 시간대·지역별 예상 순이익을 비교해 가장 적합한 업무를 동적으로 선택하는 방식이 더 적절하다.

### 기대 효과

- 차량 가동률 증가
- 유휴시간 감소
- 차량 한 대당 시간당 수익 증가
- 택시 수요 감소 시 다른 수익원으로 위험 분산
- 전체 차량 수 감소 가능성
- 자율주행차의 높은 초기 투자비 회수에 유리

## 비즈니스 문제 정의

자율주행차 운영사가 차량 한 대의 시간대·지역별 예상 순이익을 비교해 택시, 택배 또는 대기 중 가장 수익성이 높은 업무를 선택하도록 지원한다.


### 분석 단위 결정

시간 단위: 1시간
공간 단위: TLC Taxi Zone
차량 단위: For-Hire Vehicle
의사결정 단위: 시간대 × 지역


### 필요한 데이터 
#### 택시
* 시간대·지역별 승차 건수
* 평균 운행거리
* 평균 운행시간
* 평균 운임
* 시간당 예상 택시 매출
* 승객을 태우기 전 예상 대기시간
