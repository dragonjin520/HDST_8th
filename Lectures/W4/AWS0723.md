## 접근 권한

- user : AWS 사용자
- group : 사용자 모임
- role : 정책 덮어쓰기, AWS 리소스 사용
- policy : 정책(권한) 부여

### Group
- group에게 주어진 정책은 user에게 상속됨
- group간 중첩된 정책도 모두 유효함
- 
    <img src="./policy_detail.png"
     alt="Group 정책 적용"
     width="500">

### Role
- 개인 사용
    - Role을 쓰면 role에 부여된 정책 사용가능 (기존 정책 불가능)
    - Role을 벗으면 다시 개인 정책 사용 가능
- 리소스 사용
    - AWS 서비스에 접근할때 Role을 쓰고 IAM 통과
