# 0713 data engineering

## Hadoop
왜 이 프로그램이 개발되었는지 이유를 알자.
같은 일을 하는 프로그램도 각각 개발된 이유가 있고 장단점이 존재한다.

### Origin
2000년도 초반 : 구글, 페이스북, 야후 등장 - web data explosion
    - No cloud
    - server unreliable

더이상 큰 서버를 사지 말자 -> hadoop 등장

### Problem Solve
huge datasets를 cheap hardware에 저장한다.
많은 양의 데이터를 저장. 간단한 로직으로 count.

### Hadoop Layer
HDFS    -> Yarn            -> MapReduce
storage -> Resource manage -> application
완전한 층분리로 단순성 확보

Master Node (NameNode)
Slave Node (DataNode)

task parrallelism > data를 읽어와서 task 진행
data parrallelism > data가 있는 곳에서 task 진행 : 독립적 시행, 중단 후 다시 실행 가능

### HDFS
replication hardware
large data sets        
heterogeneous hardware : 성능이 서로 다른 하드웨어 집합
simple Coherency Model : WORM (write once read many) - 변하지 않는 정보 저장 (Log)
Moving Computation is cheaper than Moving Data : 단순 비용이 아니라 효용가치

### Yarn
Resource'Manager'   : AM 할당           <> Node manager : 컨테이너 하드웨어 관리
Application'Master' : 컨테이너 모니터링     <> Container

### MapReduce
Map : HDFS에서 데이터 읽어와서 chunk Split해서 mapping
        mapping한 intermediate output 뽑아서 로컬 디스크에 저장
Reduce : 로컬 디스크에서 읽어서 shuffle, sort하고 key값 별로 count


실시간으로 counting하는 것보다 log를 이용해서 사후에 counting하는게 효율적이라는 개념으로 등장.
